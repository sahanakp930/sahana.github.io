Data Structures and Algorithms Summary

1. What is an algorithm?

Ans:

An algorithm is a step-by-step procedure or formula for solving a problem. It is a sequence of instructions designed to perform a specific task or solve a problem.

2. What are data structures?

Ans:

Data structures are ways to organize and store data efficiently for access and modification. Examples include arrays, linked lists, stacks, queues, trees, and graphs.

3. Explain time complexity.

Ans:

Time complexity represents the amount of time an algorithm takes to complete as a function of the input size, typically expressed using Big-O notation, such as O(1), O(n), O(log n), etc.

4. Explain space complexity.

Ans:

Space complexity measures the total amount of memory an algorithm needs, including input data, auxiliary space, and recursion stack.

5. What is recursion?

Ans:

Recursion is a programming technique where a function calls itself to solve smaller instances of a problem until a base condition is met.

6. What are the applications of graphs?

Ans:

Graphs are used in social networks, navigation systems, web page ranking, network routing, and modeling relationships between objects.

7. What is the difference between a stack and a queue?

Ans:

A stack is a Last In, First Out (LIFO) data structure, whereas a queue is a First In, First Out (FIFO) data structure.

8. What is a binary tree?

Ans:

A binary tree is a hierarchical data structure where each node has at most two children, called the left and right children.

9. Discuss about the different studied algorithm design techniques.

Ans:

  • Divide and conquer: Breaks a problem into smaller parts and combines results. Examples: Merge Sort, Binary Search.
  • Dynamic programming: Solves overlapping subproblems, stores results. Example: Matrix Chain Multiplication.
  • Greedy algorithms: Makes the best immediate choice. Examples: Dijkstra's Algorithm, Kruskal's Algorithm.
  • Backtracking: Explores all possibilities and backtracks when needed. Examples: N-Queens, solving puzzles.
  • Branch and bound: Optimizes problem-solving with pruning. Example: Traveling Salesman Problem.
